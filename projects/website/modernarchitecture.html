The Alfresco system uses the latest technologies available for open source. These are primarily Java technologies, but we intend to make the Alfresco system available to many other languages. These technologies are on the leading edge of performance and power, but we have chosen them for their reliability and quality as well. Many of these are also implementations of standards, reference implementation of standards or de facto standards in their own right.

  • Java 1.5 including annotations and generics
    http://java.sun.com/j2se/1.5.0/docs/guide/language
  • JSR-168 Java Portlet Integration
    http://www.jcp.org/en/jsr/detail?id=168
  • Spring 1.2 Aspect-Oriented Framework
    http://www.springframework.org
  • ACEGI Aspect-Oriented Security Framework
    http://acegisecurity.sourceforge.net
  • MyFaces 1.0.9 JSF Implementation
    http://www.myfaces.org
  • Hibernate 3.0 ORM Persistence
    http://www.hibernate.org
  • Lucene 1.4 Text Search Engine
    http://lucene.apache.org
  • CIFS/SMB Microsoft File Share Protocol
    http://www.microsoft.com/mind/1196/cifs.asp
  • WebDAV / DeltaV
    http://www.webdav.org
  • JBoss Application Server 4.0
    http://www.jboss.org
  • JBoss Portal 2.0
    http://www.jboss.org/products/jbossportal
  • POI File format conversion
    http://jakarta.apache.org/poi
  • PDFBox - PDF conversion

  • MySQL

An enterprise-scale repository providing a web service API, fulltext indexing and retrieval, multiple classification schemes, team collaboration spaces and an extensible aspect framework that includes versioning, locking and content streaming.
Out-of-the-box portal solutions including portlets for repository browsing with multiple views, wizards for administration, content handling, version handling, team collaboration spaces, in-line editing of web content, desktop integration for seamless launching and saving of content.
Lean, modular component architecture plugs-in using Aspect Oriented Programming.
Faster execution paths to searching and databases.
Microsoft-compatible and CIFS-compatible file store access with automatic synchronization.
Automated content processing including auto-classification, auto-indexing and workflow handling.
Content standards support (JSR-168, WebDAV, Microsoft File System-compatible) means no vendor lock-in.
